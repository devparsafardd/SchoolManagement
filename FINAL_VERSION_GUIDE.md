# 🎉 سیستم مدیریت مدارس - نسخه نهایی (v2.0.0)

این نسخه شامل تمامی ماژول‌های مورد نیاز برای یک سیستم کامل مدیریت مدارس است.

## 🌟 ماژول‌های موجود

### مدیریت پایه
- ✅ **مدارس** (CRUD + Multi-Tenant + Excel Export)
- ✅ **کلاس‌ها** (CRUD + ظرفیت + سرپرست)
- ✅ **دانش‌آموزان** (CRUD + ثبت‌نام + پروفایل کامل)
- ✅ **اولیا** (سرپرست اصلی + حق حضانت + اجازه تحویل)
- ✅ **معلمان و کارکنان** (یک معلم در چند مدرسه + چند سمت همزمان)

### آموزشی
- ✅ **دروس** (کاتالوگ + تخصیص به پایه + تخصیص معلم به کلاس)
- ✅ **حضور و غیاب** (ثبت گروهی + ۶ وضعیت + گزارش بازه‌ای)
- ✅ **آزمون و نمره** (پشتیبانی توصیفی + عددی + نهایی‌سازی)
- ✅ **انضباطی** (تشویق/تنبیه + تاثیر در نمره انضباط)
- ✅ **کارنامه** (محاسبه خودکار معدل + خروجی PDF)

### مالی
- ✅ **شهریه و فاکتور** (انواع شهریه + پرداخت چندمرحله‌ای)
- ✅ **گزارش بدهی و وصولی**

### ارتباطات
- ✅ **اعلان‌ها** (مدرسه/کلاس/مخاطب خاص)
- ✅ **پیامک** (تک + گروهی + Log کامل)

### سیستم
- ✅ **لاگ تغییرات (Audit Log)** - ثبت خودکار همه تغییرات
- ✅ **تنظیمات سیستم** (دسته‌بندی شده)
- ✅ **پروفایل کاربر + تغییر رمز**
- ✅ **Soft Delete سراسری**
- ✅ **Serilog با خروجی فایل**

## 📋 مراحل اجرا

### قدم ۱: کپی فایل‌ها
ZIP جدید را روی پروژه فعلی Extract و Replace کنید.

### قدم ۲: Restore پکیج‌ها
راست کلیک روی Solution → **Restore NuGet Packages**

### قدم ۳: حذف دیتابیس و Migrations قبلی
به دلیل تغییرات گسترده، دیتابیس را از اول بسازید:

در SSMS:
```sql
DROP DATABASE SchoolManagement;
```

پوشه `Migrations` در `SMS.Infrastructure` را پاک کنید.

### قدم ۴: ساخت Migration
در **Package Manager Console**:
- Default Project = `SMS.Infrastructure`

```powershell
Add-Migration InitialCreate -StartupProject SMS.Web
Update-Database -StartupProject SMS.Web
```

### قدم ۵: اجرا
**F5** بزنید → ورود با `admin` / `Admin@123`

## 🎯 جریان کاری پیشنهادی

### راه‌اندازی اولیه (یکبار):
1. **مدرسه** را تعریف کنید
2. **کلاس‌ها** را اضافه کنید
3. **معلمان و کارکنان** را ثبت کنید
4. هر معلم را به مدرسه با سمت معلم تخصیص دهید
5. **دروس کلی** را اضافه کنید (یا از 26 درس پیش‌فرض استفاده کنید)
6. **دروس پایه** را تنظیم کنید (ضریب، توصیفی/عددی)

### ثبت‌نام دانش‌آموز:
1. **دانش‌آموز** اضافه کنید
2. در پروفایلش، **ولی/اولیا** را اضافه کنید
3. دانش‌آموز را به کلاس **ثبت‌نام** کنید
4. در صورت نیاز، **فاکتور شهریه** صادر کنید

### تخصیص معلم به دروس:
1. در صفحه کلاس‌ها → دکمه **📚 دروس**
2. هر درس را به یک معلم تخصیص دهید

### عملیات روزانه:
- **حضور و غیاب**: روزانه ثبت کنید
- **آزمون**: از صفحه دروس کلاس بسازید + نمره ثبت کنید
- **انضباطی**: در پروفایل دانش‌آموز ثبت کنید
- **پیامک**: اطلاع به اولیا
- **پرداخت**: ثبت پرداخت‌های شهریه

### پایان ترم:
1. آزمون‌های همه دروس را نهایی کنید
2. در **کارنامه** → کلاس را انتخاب کنید
3. دکمه **🔄 محاسبه معدل‌ها** را بزنید
4. کارنامه هر دانش‌آموز را به صورت **PDF** بگیرید

## 🐛 رفع مشکلات احتمالی

### Migration خطا داد
```powershell
Remove-Migration -StartupProject SMS.Web
# دیتابیس را DROP کنید و دوباره Add-Migration بزنید
```

### Build خطا داد
بررسی کنید:
- پکیج‌های `Microsoft.EntityFrameworkCore.Design` در `SMS.Web` موجود است
- `FrameworkReference Microsoft.AspNetCore.App` در `SMS.Infrastructure` موجود است
- نسخه `DI.Abstractions` = 8.0.2

### کارنامه PDF خطا داد
- مطمئن شوید پکیج `QuestPDF` نسخه `2024.10.2` نصب است
- `QuestPDF.Settings.License = LicenseType.Community` در Program.cs موجود است

### Soft Delete کار نمی‌کند
- موجودیت‌هایی که `ISoftDeletable` رو implement می‌کنن، خودکار Soft Delete می‌شن
- برای دیدن رکوردهای حذف شده: `db.Set<T>().IgnoreQueryFilters()`
