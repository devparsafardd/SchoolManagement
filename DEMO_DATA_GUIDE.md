# 🌱 راهنمای داده‌های نمونه (Demo Data)

این فایل لیست همه یوزرنیم/پسوردها و داده‌های نمونه‌ای رو که توسط `DemoDataSeeder` ساخته می‌شن، نشون می‌ده.

> ⚠️ **مهم:** Demo Data **فقط در محیط Development** و فقط زمانی که هیچ مدرسه‌ای در دیتابیس نیست اجرا می‌شه. در Production هرگز اجرا نمی‌شه.

## 🧪 سناریوهای تست پیشنهادی

### ۱) تست پنل ادمین
- ورود با `admin` / `Admin@123`
- بررسی `/Schools`, `/Students`, `/Staff`, `/Analytics/Index`
- مشاهده نمودارهای جامع

### ۲) تست پنل مدیر مدرسه
- ورود با `principal1` / `Pass@123` → خودکار به `/Principal/Dashboard`
- بررسی نمودار حضور و غیاب ۷ روز
- نمودار توزیع دانش‌آموزان

### ۳) تست پنل معلم
- ورود با `teacher1` / `Pass@123` → خودکار به `/Teacher/Dashboard`
- مشاهده کلاس‌های در دست تدریس
- بررسی برنامه امروز
- ورود به جزئیات یک کلاس → دیدن دانش‌آموزان و نمرات
- ثبت نمره برای آزمون
- ساخت یک تکلیف جدید

### ۴) تست پنل ولی
- پیدا کردن یک ولی از `/UserAccounts` با admin (مثلاً `parent_1_1`)
- ورود با اون یوزرنیم → داشبورد چند فرزندی
- کلیک روی فرزند → دیدن نمرات، حضور، شهریه

### ۵) تست پنل دانش‌آموز
- ورود با مثلاً `student_1_1` / `Pass@123`
- مشاهده نمرات، حضور، تکالیف
- تحویل یک تکلیف

### ۶) تست پیام‌رسان
- از هر اکانتی به `/Messages` برو
- پیام جدید بفرست به یک معلم/مدیر

## 🔄 ریست کردن دیتای دمو

اگه خواستی دیتای دمو رو دوباره از اول بسازی:

```sql
-- توی SSMS
USE master;
DROP DATABASE SchoolManagement;
```

```powershell
# توی پوشه پروژه
cd src/SMS.Web
dotnet ef database update --project ../SMS.Infrastructure --startup-project .
dotnet run
```

سیستم خودکار دیتابیس می‌سازه + داده‌های پایه (Lookup) + داده‌های دمو رو وارد می‌کنه.
